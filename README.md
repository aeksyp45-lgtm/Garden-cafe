# ຮ້ານກາເຟ ບ້ານສວນ — ລະບົບ POS + ສັ່ງອາຫານຜ່ານ QR

ໂປຣເຈັກນີ້ມີ 2 ໜ້າ:
- **ໜ້າພະນັກງານ** (`/`) — ຂາຍໜ້າຮ້ານ, ຈັດການເມນູ/ສາງ, ເບິ່ງອໍເດີຈາກລູກຄ້າ
- **ໜ້າລູກຄ້າ** (`/?order=1`) — ລູກຄ້າສະແກນ QR ແລ້ວສັ່ງອາຫານເອງຜ່ານມືຖື

ອໍເດີ ແລະ ເມນູ ຖືກເກັບໄວ້ໃນ **Supabase** (ຖານຂໍ້ມູນຄລາວຟຣີ) ເພື່ອໃຫ້ມືຖືລູກຄ້າ ແລະ tablet ພະນັກງານ ເຫັນຂໍ້ມູນດຽວກັນແບບ real-time.
ປະຫວັດການຂາຍ ແລະ ການປິດຍອດ ຍັງເກັບໄວ້ໃນ browser ຂອງພະນັກງານເທົ່ານັ້ນ (localStorage).

## ຕັ້ງຄ່າ Supabase (ເຮັດຄັ້ງດຽວ)

1. ໄປ Supabase Dashboard ຂອງໂປຣເຈັກທ່ານ → ເມນູ **SQL Editor**
2. ເປີດໄຟລ໌ `supabase-setup.sql` ໃນໂຟນເດີນີ້, ຄັດລອກທັງໝົດວາງໃສ່ SQL Editor ແລ້ວກົດ **Run**
   (ອັນນີ້ຈະສ້າງຕາຕະລາງ `orders`, `menu_config`, `sales_history`, `day_closures` ພ້ອມ Storage bucket ສຳລັບຮູບ ແລະ ໃສ່ເມນູເລີ່ມຕົ້ນໃຫ້)
3. ຄ່າ Project URL ແລະ anon key ຖືກໃສ່ໄວ້ໃນ `src/supabaseClient.js` ແລ້ວ — ບໍ່ຕ້ອງແກ້ຫຍັງເພີ່ມ

**ຫມາຍເຫດ:** ຖ້າທ່ານເຄີຍຮັນ SQL ໄຟລ໌ນີ້ມາກ່ອນແລ້ວ (ຄັ້ງທຳອິດ) ໃຫ້ຮັນຄືນອີກຄັ້ງໄດ້ເລີຍ — ໄຟລ໌ນີ້ອອກແບບໃຫ້ຮັນຊ້ຳໄດ້ (ຈະລຶບ `orders`, `menu_config`, `sales_history`, `day_closures` ເກົ່າອອກ ແລ້ວສ້າງໃໝ່ໝົດ) ⚠️ ຂໍ້ມູນອໍເດີ/ເມນູ/ປະຫວັດເກົ່າຈະຫາຍໄປ ຖ້າຕ້ອງການເກັບໄວ້ ໃຫ້ສຳຮອງກ່ອນ ຫຼືບອກໄດ້ໃຫ້ຂ້ຽນ SQL ແບບບໍ່ລຶບຂໍ້ມູນເກົ່າ

## ວິທີໃຊ້ຢູ່ໃນຄອມພິວເຕີຂອງທ່ານ (ທົດລອງກ່ອນ)

ຕ້ອງຕິດຕັ້ງ Node.js ກ່ອນ (ດາວໂຫລດຟຣີໄດ້ທີ່ https://nodejs.org — ເລືອກເວີຊັນ LTS)

ເປີດ Terminal ໃນໂຟນເດີນີ້ ແລ້ວພິມ:

```
npm install
npm run dev
```

- ໜ້າພະນັກງານ: ເປີດລິ້ງທີ່ຂຶ້ນມາ (ເຊັ່ນ http://localhost:5173)
- ໜ້າລູກຄ້າ (ທົດລອງ): ເປີດລິ້ງດຽວກັນ ແຕ່ເພີ່ມ `?order=1` ຕໍ່ທ້າຍ (ເຊັ່ນ http://localhost:5173/?order=1)

ລອງເປີດ 2 ແທັບ (ໜ້າພະນັກງານ 1 ແທັບ, ໜ້າລູກຄ້າ 1 ແທັບ) — ສັ່ງອາຫານຈາກແທັບລູກຄ້າ ຈະຂຶ້ນຢູ່ແທັບພະນັກງານທັນທີໃນແທັບ "ອໍເດີລູກຄ້າ"

## ວິທີເອົາໄປໃຊ້ຈິງທີ່ຮ້ານ (Deploy)

### ຂັ້ນທີ 1 — ອັບໂຄ້ດຂຶ້ນ GitHub
1. ສ້າງບັນຊີຟຣີທີ່ https://github.com
2. ສ້າງ repository ໃໝ່ ແລ້ວອັບໂຟນເດີນີ້ຂຶ້ນໄປ (ໃຊ້ GitHub Desktop ຖ້າຍັງບໍ່ຄຸ້ນກັບ git command)

### ຂັ້ນທີ 2 — Deploy ຜ່ານ Vercel (ຟຣີ)
1. ສະໝັກບັນຊີທີ່ https://vercel.com ໂດຍໃຊ້ບັນຊີ GitHub ດຽວກັນ
2. ກົດ "New Project" ແລ້ວເລືອກ repository ທີ່ອັບໄວ້
3. ກົດ Deploy — Vercel ຈະສ້າງລິ້ງເວັບໄຊທ໌ໃຫ້ອັດຕະໂນມັດ (ເຊັ່ນ `cafe-pos-xxxx.vercel.app`)

### ຂັ້ນທີ 3 — ພິມ QR ວາງໃສ່ໂຕະ
1. ເປີດລິ້ງທີ່ deploy ແລ້ວ ໄປທີ່ແທັບ "ອໍເດີລູກຄ້າ" ໃນໜ້າພະນັກງານ
2. ຈະເຫັນ QR code ຢູ່ເທິງສຸດ — ອັນນີ້ຊີ້ໄປຫາລິ້ງ `<ລິ້ງເວັບໄຊທ໌ຂອງທ່ານ>/?order=1` ໂດຍອັດຕະໂນມັດ
3. ຖ່າຍໜ້າຈໍ ຫຼື print QR ນີ້ອອກມາ ວາງໄວ້ຢູ່ແຕ່ລະໂຕະ

## ວິທີໃຊ້ຢູ່ຮ້ານ (ພະນັກງານ)

1. ເປີດລິ້ງເວັບໄຊທ໌ຢູ່ tablet — ນີ້ຄືໜ້າຂາຍປົກກະຕິ
2. ໃນ browser ມືຖື/tablet, ເລືອກ "Add to Home Screen" ເພື່ອໃຫ້ມີໄອຄອນຄືແອັບ
3. ເມື່ອລູກຄ້າສັ່ງຜ່ານ QR, ອໍເດີຈະຂຶ້ນຢູ່ແທັບ "ອໍເດີລູກຄ້າ" ພາຍໃຕ້ "ອໍເດີທີ່ລໍຖ້າ" — ກົດ **"ຮັບອໍເດີ"** ເມື່ອເລີ່ມກະກຽມ (ຍັງບໍ່ຄິດເງິນ)
4. ໂຕະນັ້ນຈະຂຶ້ນຢູ່ໃນ **"ໂຕະທີ່ຍັງບໍ່ຈ່າຍ"** — ຖ້າລູກຄ້າສັ່ງເພີ່ມ (ຮອບ 2, 3, ...) ອໍເດີໃໝ່ຈະລວມເຂົ້າກັບໂຕະດຽວກັນອັດຕະໂນມັດ
5. ເມື່ອລູກຄ້າພ້ອມຈ່າຍ, ກົດ **"ຈ່າຍເງິນ"** ຢູ່ໂຕະນັ້ນ — ລາຍການທັງໝົດ (ທຸກຮອບ) ຈະຖືກໂຫຼດເຂົ້າ POS ໃຫ້ຄິດເງິນຄັ້ງດຽວ

## ວິທີໃສ່ຮູບອາຫານໃນເມນູ

ໃນໜ້າ "ຈັດການເມນູ & ສາງ" ແຕ່ລະລາຍການເມນູຈະມີ:
- ປຸ່ມ **"📷 ອັບໂຫລດ"** — ເລືອກຮູບຈາກຄອມ/ມືຖືໂດຍກົງ, ອັບໂຫລດເຂົ້າ Supabase Storage ໃຫ້ອັດຕະໂນມັດ (ບໍ່ຕ້ອງໄປຫາລິ້ງຈາກເວັບໄຊທ໌ອື່ນ)
- ຫຼືວາງລິ້ງຮູບເອງໄດ້ຄືເກົ່າ (ຖ້າມີຮູບຢູ່ອອນລາຍແລ້ວ)

ຮູບຈະຂຶ້ນທັນທີທັງໃນໜ້າ POS ແລະ ໜ້າສັ່ງອາຫານຂອງລູກຄ້າ

## ວິທີສ້າງ QR ຕາມໂຕະ

ໃນແທັບ "ອໍເດີລູກຄ້າ" ໃສ່ຈຳນວນໂຕະ (ເຊັ່ນ 10) — ລະບົບຈະສ້າງ QR ໃຫ້ແຕ່ລະໂຕະອັດຕະໂນມັດ ຄລິກຂວາໃສ່ QR ແຕ່ລະຮູບ → "ບັນທຶກຮູບ" ເພື່ອເອົາໄປພິມວາງໃສ່ໂຕະ ເມື່ອລູກຄ້າສະແກນ QR ຂອງໂຕະນັ້ນ, ເລກໂຕະຈະຖືກໃສ່ໃຫ້ອັດຕະໂນມັດ (ບໍ່ຕ້ອງພິມເອງ)

## ຂໍ້ມູນທັງໝົດ Backup ຢູ່ Supabase (ຈຸດດຽວ)

ຕອນນີ້ **ທຸກຢ່າງ** ເກັບໄວ້ໃນ Supabase ແລ້ວ — ບໍ່ໄດ້ອີງໃສ່ browser ຂອງອຸປະກອນໃດອຸປະກອນໜຶ່ງອີກຕໍ່ໄປ:
- ເມນູ (`menu_config`)
- ອໍເດີຈາກລູກຄ້າ (`orders`)
- ປະຫວັດການຂາຍ (`sales_history`)
- ການປິດຍອດປະຈຳວັນ (`day_closures`)
- ຮູບເມນູ (Supabase Storage bucket `menu-images`)

ນັ້ນໝາຍຄວາມວ່າ:
- ຖ້າລ້າງ browser cache ຫຼືປ່ຽນ tablet ໃໝ່, **ຂໍ້ມູນຈະບໍ່ຫາຍ**
- ໃຊ້ຫຼາຍ tablet ພ້ອມກັນໄດ້ ໂດຍເຫັນຂໍ້ມູນດຽວກັນ
- ຢາກເບິ່ງ/ດາວໂຫລດຂໍ້ມູນດິບ ສາມາດເຂົ້າ Supabase Dashboard → Table Editor ໄດ້ທຸກເວລາ

## ຂໍ້ຄວນຮູ້ (ສຳຄັນ)

- **ທຸກຂໍ້ມູນຢູ່ Supabase** — ເມນູ, ອໍເດີ, ປະຫວັດການຂາຍ, ການປິດຍອດ ແລະ ຮູບ — ຫຼາຍອຸປະກອນເຫັນຂໍ້ມູນດຽວກັນ, ອັບເດດ real-time, ບໍ່ຫາຍເມື່ອລ້າງ browser cache
- **ໂຄງສ້າງໄຟລ໌**: ແກ້ໄຂເມນູ/ລາຄາ/ຮູບໄດ້ທີ່ໜ້າ "ຈັດການເມນູ & ສາງ" ໃນແອັບ (ບໍ່ຕ້ອງແກ້ໂຄ້ດ)
- **anon key** ໃນ `supabaseClient.js` ອອກແບບມາໃຫ້ເປີດເຜີຍໄດ້ (ໃຊ້ຄູ່ກັບ Row Level Security policies) — ແຕ່ຢ່າເອົາ **service_role key** ຫຼືລະຫັດຜ່ານຖານຂໍ້ມູນມາໃສ່ໂຄ້ດເດັດຂາດ
- ນະໂຍບາຍ RLS ປັດຈຸບັນເປີດໃຫ້ "anon" (ໃຜກໍໄດ້ທີ່ຮູ້ URL) ອ່ານ/ຂຽນໄດ້ໝົດ — ເໝາະສຳລັບຮ້ານນ້ອຍທີ່ໃຊ້ພາຍໃນ, ຖ້າຢາກເພີ່ມການລ໊ອກອິນສຳລັບພະນັກງານໃນອະນາຄົດ ບອກໄດ້
